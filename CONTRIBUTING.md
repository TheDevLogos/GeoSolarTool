# Contributing to GeoSolarTool

Thanks for your interest in improving GeoSolarTool.

## Code of conduct

Be respectful, constructive, and collaborative. Keep discussions focused on product quality, maintainability, and user value.

## How to contribute

1. Fork the repository.
2. Create a branch for your change.
3. Make your edits.
4. Test locally.
5. Open a pull request with a clear description.

## Suggested workflow

```bash
git checkout -b feat/your-change
```

Commit messages should be clear and specific.

## What we welcome

- Bug fixes.
- Documentation improvements.
- UX refinements.
- Performance optimizations.
- New analysis modules.
- Better PDF export flows.
- Improvements for static deployment.

## Before submitting

Please verify that your change:

- Does not break the current UI layout.
- Preserves the static deployment model.
- Keeps map rendering stable.
- Maintains the existing product logic.
- Includes documentation when needed.

## Testing guidance

If you modify the app, confirm at minimum:

- The map loads correctly.
- The main analysis panel renders.
- Area drawing still works.
- PDF export still works.
- The app works on Vercel as a static site.
